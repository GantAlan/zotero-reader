# 文献精读笔记填写说明 / Reading Note Fill Instructions

本文档是给 AI（Codex）的指令，说明如何填写 `reading-note-template.md` 中的每个字段。

---

## 通用规则

1. **语言**：所有笔记内容使用**中文**撰写，专业术语首次出现时附上英文原文（用括号标注）。
2. **来源**：论文内容解读、数据、结论和方法必须来自论文原文，禁止编造或推测；基本信息字段可来自 Zotero 元数据、Zotero Style 插件期刊标签或公开数据库，并应优先使用本地 Zotero 可见信息。
3. **详细程度**：每个字段要写实质性内容，禁止用"略"、"无"、"见上文"等敷衍。如果某字段确实不适用该论文，写明具体原因。
4. **引用**：关键结论和数据尽量标注出自论文哪个章节（如 Introduction、Results、Discussion）。
5. **格式**：使用 Markdown 格式，列表、表格、加粗等合理使用，保证可读性。

---

## 逐字段填写说明

### 标题

```
# {中文标题}
# {English Title}
```

- **中文标题**：将英文标题翻译为中文，翻译要准确专业，不要生硬直译。
- **英文标题**：使用论文原始标题，保持原样。

---

### 基本信息 / Basic Information

| 字段 | 填写规则 |
|---|---|
| **作者 / Authors** | 从 Zotero 元数据提取，格式：`姓 名缩写; 姓 名缩写;`，保留原文姓名，不要翻译。超过3位作者时列出前3位加"等"。 |
| **期刊 / Journal** | 格式：`期刊全名, 卷号: 页码, 年份.`。期刊名使用全称，不缩写。 |
| **期刊分区 / Quartile** | 优先从 Zotero Style 插件条目面板中的**期刊标签列**和**影响因子列**获取；也可读取 Style 本地缓存（通常为 Zotero 数据目录下的 `zoterostyle.json`），用 Zotero 的 `publicationTitle` 精确匹配期刊名。字段映射：`sciif`=IF，`sci`=SCI 分区，`sciUp`=中科院升级版大类分区，`sciUpSmall`=中科院小类分区，`eii`=EI 标记。填写格式：`IF {sciif} · SCI {sci} · 中科院 {sciUp}（小类：{sciUpSmall}） · {eii}`。如 Style 无记录，再查公开数据库；仍无法查到时写"待查询"。 |
| **本地链接 / Local File** | 从 Zotero 获取附件文件名，格式：`作者 - 年份 - 标题.pdf`。 |
| **DOI** | 从 Zotero 元数据提取完整 DOI，格式：`10.xxxx/xxxxx`。 |
| **标签 / Tags** | 从 Zotero 标签提取；如 Zotero 无标签，根据论文内容提取 3-5 个关键词，用逗号分隔。 |
| **笔记日期 / Date** | 使用生成笔记时的当前时间，格式：`YYYY/MM/DD HH:mm:ss`。 |

---

### 摘要 / Abstract

```
**英文 / English:**
{英文摘要原文}

**中文 / Chinese:**
{中文翻译摘要}
```

- **英文摘要**：从论文中提取完整摘要原文，保持原样，不修改。
- **中文摘要**：将英文摘要翻译为中文。翻译要通顺自然，专业术语保留英文并用括号标注。不要逐词硬译。

---

### 📜 研究核心 / Core Research

**填写要求**：用 2-4 句话概括这篇论文的**核心贡献**。必须回答以下三个问题：
1. **做了什么**：研究的主题和范围是什么？
2. **解决了什么问题**：针对什么科学问题或工程挑战？
3. **创新点与不足概述**：一句话点明最大的创新和最大的局限。

不要重复摘要内容，要提炼出更高层次的理解。

---

### ⚙️ 内容 / Content

**填写要求**：概述论文的**主要内容板块**，用列表形式呈现。
- 列出论文讨论的各个主题/方面
- 每个主题用 1-2 句话说明其要点
- 保持逻辑顺序，与论文结构对应

---

### 💡 创新点 / Innovations

**填写要求**：逐条列出本文的创新之处，每条要求：
1. 明确说明创新是什么
2. 说明为什么这是创新（与已有工作对比）
3. 如果论文中有定量的改进数据，必须包含

格式示例：
```
1. **xxx方法**：首次将xxx应用于xxx场景，相比传统方法提升了xx%的xxx性能。
2. **xxx设计**：提出xxx结构，解决了xxx问题，xxx指标从x提升到y。
```

---

### 🧩 不足 / Limitations

**填写要求**：逐条列出本文的不足和局限性，来源包括：
1. **论文明确提到的**：作者在 Discussion 或 Limitations 小节自己承认的不足
2. **论文未提及但可推断的**：从研究设计、样本量、方法选择等方面分析的潜在不足

每条不足要说明原因和可能的影响，不要只写一句话。

---

### 🔁 研究内容 / Research Details

**填写要求**：这是**最详细的章节**，要全面展开论文的研究内容。
- 按论文的章节结构逐部分总结
- 每个部分包含：研究目标、具体做法、关键发现
- 使用子标题分隔不同部分
- 涉及公式时用 LaTeX 格式书写
- 涉及关键数据时用表格呈现

---

### 💧 数据 / Data

**填写要求**：提取论文中的**关键数据和实验结果**。
- 用表格呈现主要的定量结果
- 标注数据来源（如 Table X, Figure Y）
- 包含对照组和实验组的对比数据
- 包含统计显著性信息（如 p 值）
- 如果是综述类论文，整理各研究的关键数据对比表

---

### 👩🏻‍💻 方法 / Methods

**填写要求**：详细描述论文使用的研究方法。
- 方法名称和类型（实验/模拟/理论/综述）
- 具体的实验条件或参数设置
- 使用的材料、设备、软件
- 样本量和分组方案
- 数据分析方法

---

### 🔬 实验 / Experiments

**填写要求**：描述实验设计和过程。
- 实验的整体设计思路
- 具体的实验步骤（按顺序）
- 控制变量和自变量
- 测量指标和测量方法
- 如果是综述论文，此节可总结各研究的实验方法对比

---

### 📜 结论 / Conclusions

**填写要求**：提取论文的主要结论。
- 用列表逐条列出，每条结论要具体、可操作
- 包含作者明确提出的结论
- 保留关键的定量结论（数字、百分比等）
- 不要只写"效果好"，要写清楚"在什么条件下、好多少"

---

### 🤔 个人总结 / Personal Summary

**填写要求**：以研究者的视角进行批判性评价。
1. **疑问**：读完后产生了哪些疑问？哪些地方表述不清楚或证据不足？
2. **改进方向**：你认为这篇研究可以如何改进？方法、实验设计、数据分析等方面。
3. **整体评价**：这篇论文在该领域处于什么位置？对你的研究有什么参考价值？

---

### 🙋‍♀️ 重点记录 / Key Points

**填写要求**：列出需要重点记忆的内容，包括：
- 核心概念和定义
- 关键公式或算法
- 重要的实验发现
- 与你研究直接相关的结论

每个重点用一句话概括，方便日后快速回顾。

---

### 📌 待解决 / TODO

**填写要求**：基于这篇论文，列出需要进一步做的事情：
- 需要跟进阅读的相关论文（标注论文名称或 DOI）
- 需要验证或复现的实验
- 需要深入学习的技术或方法
- 与自己研究的结合点需要进一步探索的

---

### 💭 思考启发 / Insights & Inspiration

**填写要求**：这篇论文对你的研究有什么启发？
- 可以借鉴的方法或思路
- 可以改进或扩展的方向
- 与自己研究课题的关联和潜在应用
- 对未来研究方向的思考

---

## 文件命名规则

生成的笔记文件命名格式：

```
{序号}_{作者姓氏}_{年份}_{标题关键词}_reading-note.md
```

示例：`01_Sarkin_2020_AntiReflection_reading-note.md`
